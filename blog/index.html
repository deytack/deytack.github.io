デザイナーが、自分用のブログをイチから作る。 Dec 23

はじめに

Middleman でビルドした記事を Github Pages にアップする形式のブログを作りたい。

このようなケースの時にとてもおすすめできる。

  • お気に入りのエディタで記事を書きたい
    • Word Press 等の著名 CMS や、各種ブログサービスには便利な UI や管理画面が付いているが、 メモ感覚で使うよりガッツリ「書く」事を要求されているようでつらい
  • デザインは自分でゼロから作りたい
    • 初期の段階では把握できないくらい多くのテンプレートがある状況は好ましくない
    • フロントに出る部分を自分ですべて把握したい
    • 開発とビルドされた状態とを分離したい。ついでに Alt JS やその他便利なテンプレート言語を使いたい
  • ミニマムの機能だけあればいい
    • コメント、フィードバック、その他動的に変化する要素はいらない
    • 何か機能が必要だったら、それに応じてサービスのウィジェットや提供されている API を組み込んで対応したい
    • レスポンスはできるだけ早くしたい

Middleman と Github Pages について

Middleman は、Ruby ベースで書かれた静的サイトジェネレータ。 ローカルでページをビルドするところに特徴がある。 ページ数 2 桁くらいの Web サイトを統一されたデザインで作りたい場合に便利そうだ。

Github Pages は、Github に username.github.io というリポジトリを作ることで、 そのリポジトリ内のファイルを Web サイトとして閲覧可能な仕組み。 PHP や Ruby は動かない。
